# 钉钉考勤技能发布说明

## 目标

这份文档用于发布和交付 `dingtalk-attendance` skill，说明使用前提、支持范围、命令入口、状态码和故障排查方式。

## 使用前提

- 已安装 `adb`
- 手机已开启 USB 调试
- 手机已授权当前电脑的 `adb`
- 已安装 Python 3
- 本地 OCR 可选：
  - macOS：`RapidOCR + OpenCV`
  - Windows / Linux：`PaddleOCR + OpenCV`

## 支持范围

### 已支持

- 打开钉钉
- 自动切到工作台
- 滑动到工作台顶部
- 打开考勤打卡页面
- 处理常见关闭弹窗、暂不更新、误入页面返回
- 本地 OCR 失败后转模型兜底
- 锁屏解锁：
  - 无锁屏
  - 数字 PIN 锁屏

### 明确不支持

- 图案锁
- 字母数字密码
- 厂商定制复杂锁屏

遇到不支持锁屏时，脚本会返回 `unsupported_lock_type`，不会盲试。

## 默认安全策略

- `clock-in` / `clock-out` 默认是 `dry-run`
- 默认只识别按钮，不真实点击
- 只有显式加 `--execute` 才允许真实执行
- 本地 OCR 依赖安装失败后，会记住 `model_fallback`
- 本地 OCR 本轮识别失败时，会直接返回 `needs_model_input`

## 常用命令

### 页面导航

```bash
python3 scripts/run.py open --json
python3 scripts/run.py open-step --json
python3 scripts/run.py status --json
```

### 测试模式打卡

```bash
python3 scripts/run.py clock-in --json
python3 scripts/run.py clock-out --json
```

### 真实执行

```bash
python3 scripts/run.py clock-in --execute --json
python3 scripts/run.py clock-out --execute --json
```

### 配置与密码缓存

```bash
python3 scripts/run.py show-config --json
python3 scripts/run.py set-lock-password --password 000000 --json
python3 scripts/run.py clear-lock-password --json
```

指定设备：

```bash
python3 scripts/run.py set-lock-password --serial <serial> --password 000000 --json
python3 scripts/run.py clear-lock-password --serial <serial> --json
```

### 本地 OCR 恢复

```bash
python3 scripts/run.py retry-local-install --json
```

## 配置说明

运行时配置位于：

```bash
.runtime/config.json
```

关键字段：

- `mode`：`local` 或 `model_fallback`
- `dry_run`：默认 `true`
- `last_device_serial`：最近一次使用的设备
- `devices`：设备级运行状态

说明：

- 锁屏密码缓存按设备 `serial` 隔离
- `show-config` 只返回掩码，不直接回显明文密码
- 公开发布版不再把锁屏密码写入 `config.json`
- macOS 使用 Keychain，Windows 使用 Credential Manager，Linux 使用 Secret Service
- 如果系统没有可用安全存储能力，`set-lock-password` 会报错，不会降级到明文缓存

旧版本迁移：

- 如果历史版本曾把锁屏密码写入本地文件，升级后脚本会在首次读取时尝试迁移到系统安全存储
- 迁移成功后，会清理本地文件中的旧密码字段

## 关键状态码

- `attendance_ready`：已进入考勤页
- `dry_run_ready`：已识别到按钮，但当前是测试模式，没有真实点击
- `completed`：已完成真实点击并通过页面复检
- `already_done`：当前页面显示已经打过对应卡
- `needs_confirmation`：通常是下班卡已打过，需要用户确认是否再次执行
- `needs_unlock_password`：设备锁屏且当前没有可用数字 PIN 密码
- `unlock_password_invalid`：密码校验失败，当前设备缓存已清空
- `unsupported_lock_type`：当前锁屏不是数字 PIN
- `needs_model_input`：需要模型查看截图并做一步判断
- `config`：已返回当前配置
- `lock_password_updated`：已更新当前设备密码缓存
- `lock_password_cleared`：已清除当前设备密码缓存

## 常见 fallback_reason

- `local_ocr_returned_no_text_boxes`
  - 含义：本地 OCR 没识别到文本
  - 处理：直接使用模型兜底

- `navigation_target_not_found_locally`
  - 含义：本地识别没找到工作台或考勤入口
  - 处理：查看截图，优先判断返回、关闭弹窗、工作台、考勤入口

- `attendance_action_not_found_locally`
  - 含义：已到考勤页，但本地识别没确认上班/下班按钮
  - 处理：查看截图，确认是“上班打卡”“下班打卡”还是“已打卡”

- `open_step_requires_model_judgement`
  - 含义：当前页面需要模型做一步判断
  - 处理：只执行一步，不连续盲点

- `navigation_retry_exhausted`
  - 含义：自动导航重试次数过多
  - 处理：人工查看截图，判断是否有异常弹窗或误入页面

## 故障排查

### `adb` 没设备

检查：

- `adb devices -l`
- 手机是否已开启 USB 调试
- 手机是否已授权电脑

### 一直返回 `needs_unlock_password`

检查：

- 是否已经传入 `DINGTALK_ATTENDANCE_LOCK_PASSWORD`
- 是否使用的是数字 PIN
- 是否要先用 `set-lock-password` 写入缓存

### 返回 `unsupported_lock_type`

说明当前锁屏不是数字 PIN，需切换为数字 PIN 后再使用自动解锁。

### 一直进入模型兜底

检查：

- 当前页面是否有广告弹窗或更新弹窗
- 本地 OCR 依赖是否可用
- 是否需要运行 `retry-local-install --json`

### 配置状态不清楚

先执行：

```bash
python3 scripts/run.py show-config --json
```

## 发布建议

- 默认保留 `dry_run: true`
- 默认引导用户先跑 `open --json` 和 `clock-out --json`
- 只有在用户确认真实执行时，才提示使用 `--execute`
- 公开发布时，优先检查目标系统是否具备安全存储能力
