# 钉钉考勤多 CLI 发布摘要

本仓库已经从单一 Codex skill 调整为“核心共用 + 多 CLI 适配层”结构。

## 当前支持

- Codex
- OpenCode
- Claude Code
- Gemini CLI
- OpenClaw

## 核心目录

- `core/dingtalk-attendance`
  - 核心脚本、测试、状态机参考说明
- `docs/protocol.md`
  - 稳定 JSON 协议
- `docs/publishing.md`
  - 详细发布与安装策略
- `adapters/`
  - 各 CLI 的入口文档

## 发布原则

- 默认保持 `dry_run`
- 锁屏只支持无锁屏和数字 PIN
- 本地 OCR 失败后直接进入模型兜底协议
- 适配层只负责触发与协议消费，不复制核心脚本

## 建议阅读顺序

1. 先看 `README.md`
2. 再看 `docs/protocol.md`
3. 按目标平台选择 `adapters/` 下对应入口文档
4. 需要发布或分发时查看 `docs/publishing.md`
